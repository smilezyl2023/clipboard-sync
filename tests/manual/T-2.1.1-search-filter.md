## M-1 搜索框外观与交互（桌面端）

- **关联验收标准**: spec §验收标准 / §边界条件
- **操作步骤**:
  1. 打开页面，查看历史记录区域
  2. 检查搜索框是否在 section-header 下方
  3. 查看搜索框是否有搜索图标和 placeholder 文字
  4. 输入关键词，查看清除按钮是否出现
  5. 点击清除按钮，输入框清空
- ✅ **成功表现**:
  - 搜索框在 header 下方、记录列表上方
  - 左侧有搜索图标（放大镜）
  - placeholder 为"搜索历史记录..."
  - 输入文字后右侧出现 X 清除按钮
  - 点击 X 后输入框清空，清除按钮消失
- ❌ **失败表现**:
  - 搜索框位置不对
  - placeholder 文字缺失或错误
  - 清除按钮不出现或不工作

## M-3 键盘无障碍

- **关联验收标准**: spec §验收标准 — 无障碍
- **操作步骤**:
  1. Tab 键聚焦搜索框
  2. 输入文字
  3. Tab 键移至清除按钮
  4. Enter 键点击清除按钮
  5. 检查焦点样式是否可见
- ✅ **成功表现**:
  - Tab 键可按顺序聚焦搜索框 → 清除按钮
  - 焦点有可见的蓝色 outline
  - Enter 键可激活清除按钮
  - 清空后焦点回到搜索框（或合理位置）
- ❌ **失败表现**:
  - Tab 键跳过了搜索框或清除按钮
  - 焦点样式不可见
  - Enter 键无法激活清除按钮

## M-4 上传中记录在搜索时仍可见

- **关联验收标准**: spec §边界条件
- **操作步骤**:
  1. 上传一个大文件（触发 uploading 状态）
  2. 在搜索框输入与上传文件名无关的关键词

## M-5 回归：核心功能不受影响

- **关联验收标准**: spec §回归
- **操作步骤**:
  1. 同步一段文本 → 确认记录出现
  2. 点击文本记录加载回输入框
  3. 上传一张图片 → 确认上传完成
  4. 删除一条记录 → 确认撤销 Toast 出现
  5. 打开图片 lightbox → ESC 关闭

- ✅ **成功表现**:
  - 以上所有功能正常工作
  - 搜索过滤与这些功能不冲突
- ❌ **失败表现**:
