## M-1 深色模式下所有组件颜色正确

- **关联验收标准**: spec §验收标准 "深色模式下所有组件颜色正确（包括 Toast、lightbox、batch-bar、auth-overlay）"
- **命中手测条件**: #1 视觉/动画体感 — 需要人眼判断颜色和对比度是否合理
- **操作步骤**:
  1. 确保系统处于深色模式
  2. 打开页面 http://127.0.0.1:3000
  3. 检查以下区域：
     - 登录弹窗（auth-overlay）：背景渐变色、卡片背景、输入框、按钮
     - header 区域：brand-icon 背景、标题文字、用户 chip
     - 输入区域（input-card）：textarea 背景、边框、placeholder 文字
     - 搜索筛选：搜索框、类型标签（选中/未选中）、清除按钮
     - 记录列表：记录卡片、文字颜色、元数据区域
     - 图片/文件缩略图和图标
     - Toast 通知（触发方式：同步一条文本后查看）
     - batch-bar 批量操作栏
     - lightbox 图片预览（点击缩略图）
     - 骨架加载动画
     - 错误横幅（如有）
     - 空状态提示
     - 离线横幅
  4. 尝试切换回浅色模式，确认浅色不受影响
- ✅ **成功表现**（必须全部满足）:
  - 所有文字清晰可读，对比度充足
  - 卡片、边框、背景层次分明，不会融为一体
  - 主色调（蓝色）在深色背景下仍然醒目
  - 危险色（红色）在深色背景下仍然醒目
  - Toast 在深色背景下明显可见
  - 浅色模式完全不受影响
- ❌ **失败表现**（出现任一即视为不通过）:
  - 任何区域文字因对比度不足而难以阅读
  - 颜色出现刺眼的异常色块
  - 浅色主题被意外改变

**测试结果**: [x] 通过  [ ] 失败
